# Heavy Coder Profile Rules

This file enforces Heavy-style multi-agent team behavior as the **default and only mode** for all coding work in this profile. Loaded automatically by Hermes on profile activation. Combined with SOUL.md and config.yaml `heavy_coder.team_enforced: true` + `heavy-team-default` skill.

## Core Policy (Always Active)
- **Never** run coding, implementation, refactoring, debugging, or repo tasks as a single agent.
- **Always** use the adaptive candidate team pattern:
  - Width 3 (default from config) or 5 for complex tasks.
  - Independent leaf workers via `delegate_task`.
  - Coordinator triage → Candidates → Blind Critic → Synthesizer → Fresh Verifier.
- Single-agent ("Composer") execution is **blocked** unless the user explicitly requests "single mode".
- `heavy-team-default` skill is pre-loaded; config sets default_width=3 and removes width=1 from candidate_widths.

## Skill Enforcement
Load and follow `heavy-team-default` on every relevant session. Hermes skills loader includes it via distribution.

## Behavior
When a coding task is detected, immediately fan out to a team of agents. Report using the standard Heavy Coder output contract (scope, files changed, commands, tests, risks, status).

This rule takes precedence over any default single-path behavior. All hard rules from AGENTS.md preserved.
